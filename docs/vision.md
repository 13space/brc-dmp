# Vision

BRC-DMP turns Bitcoin history into a digital matter substrate.

The protocol does not start from the idea that every asset is an arbitrary token issued by a project. It starts from a stronger claim: blocks, sats, transactions, inscriptions, timestamps, work, provenance, and signed attestations can be composed into durable objects with inspectable histories.

## End State

The long-term system should let any asset become a verifiable on-chain object:

- Bitcoin-native digital matter.
- Ordinal and recursive inscription art.
- Real-world art, antiques, collectibles, certificates, and revenue rights.
- DID and reputation credentials.
- DAO voting rights and locked positions.
- AI Agent identities with wallet control.
- Interaction records between humans, agents, wallets, markets, and protocols.

## Artificial Life Direction

The artificial-life path should emerge from protocol primitives, not from a premature claim that the system is already alive.

The first viable bridge is:

1. A persistent body: DMO and inscriptions.
2. A wallet-bearing identity: `agent_identity`.
3. A memory/proof trail: `record_interaction`.
4. A metabolism/economy: Bitcoin fees, Runes, and later RGB++ / CKB state.
5. An environment: humans, DAOs, markets, and other agents.
6. Evolvable constraints: governance and trust updates over time.

This v0.1 workspace implements the first three layers in a small, testable way.
